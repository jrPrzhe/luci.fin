# Использование SQLite на Railway (для тестирования)

⚠️ **ВНИМАНИЕ:** SQLite на Railway не рекомендуется для production, так как:
- Данные теряются при перезапуске контейнера
- Нет постоянного хранилища (по умолчанию)
- Нет возможности масштабирования

## Быстрое решение (для тестирования)

Если вы хотите использовать SQLite на Railway (как локально):

### Шаг 1: Удалите или очистите переменную DATABASE_URL

1. Откройте ваш Backend сервис на Railway
2. Перейдите в **Variables**
3. **Найдите переменную `DATABASE_URL`**
4. **УДАЛИТЕ её** или установите значение `sqlite:///./finance.db`

**Важно:** Если `DATABASE_URL` установлен с неправильным значением (например, `postgresql://localhost:5432/...`), приложение будет пытаться подключиться к PostgreSQL на localhost, что вызовет ошибку.

### Шаг 2: Убедитесь, что SQLite указан явно

1. В **Variables** добавьте или проверьте:
   ```
   DATABASE_URL=sqlite:///./finance.db
   ```

### Шаг 3: Перезапустите деплой

Railway автоматически перезапустит после изменения переменных.

### Шаг 4: Проверьте логи

После перезапуска приложение должно:
- Создать файл `finance.db` в рабочей директории
- Успешно запуститься

## Ограничения SQLite на Railway

❌ **Проблемы:**
- Данные теряются при каждом перезапуске контейнера
- Нет постоянного хранилища между деплоями
- Нельзя использовать для production

✅ **Подходит для:**
- Тестирования
- Демонстрации
- Временных проектов

## Рекомендация: Переход на PostgreSQL

Для production лучше использовать PostgreSQL:

1. Добавьте PostgreSQL сервис на Railway (см. `RAILWAY_DATABASE_FIX.md`)
2. Railway автоматически создаст правильный `DATABASE_URL`
3. Ваше приложение автоматически переключится на PostgreSQL
