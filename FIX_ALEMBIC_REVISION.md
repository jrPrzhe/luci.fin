# Исправление проблемы с миграциями Alembic

## Проблема

Alembic не может найти ревизию `a920226c4581`, которая есть в БД, но отсутствует в файлах миграций.

## Решение

### Вариант 1: Проверить текущую ревизию в БД

```bash
psql -h 195.43.142.121 -U finance_user -d finance_db -c "SELECT * FROM alembic_version;"
```

### Вариант 2: Обновить ревизию на правильную

1. Найдите последнюю миграцию в `backend/alembic/versions/`
2. Обновите ревизию в БД:

```bash
psql -h 195.43.142.121 -U finance_user -d finance_db -c "UPDATE alembic_version SET version_num = 'ПОСЛЕДНЯЯ_РЕВИЗИЯ';"
```

### Вариант 3: Удалить запись и пересоздать

```bash
psql -h 195.43.142.121 -U finance_user -d finance_db -c "DELETE FROM alembic_version;"
```

Затем при следующем запуске Alembic создаст новую запись.

### Вариант 4: Пропустить миграции (если схема БД уже актуальна)

Добавьте в Railway переменную окружения:
- `SKIP_MIGRATIONS=true`

## Проверка

После исправления приложение должно запускаться без предупреждений о миграциях.
