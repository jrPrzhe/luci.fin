# Информация о сервере базы данных

## Конфигурация сервера

- **IP-адрес**: 195.43.142.121
- **ОС**: Debian 11
- **CPU**: 1x2.2ГГц
- **RAM**: 0.5ГБ (512МБ) ⚠️ **ОГРАНИЧЕННЫЕ РЕСУРСЫ**
- **Диск**: 10ГБ HDD RAID
- **SSH**: Готов к использованию

## ⚠️ Важные ограничения

### Память (0.5ГБ RAM)

Сервер имеет очень ограниченный объем памяти. Это означает:

1. **PostgreSQL должен быть оптимизирован** - см. раздел 3.5 в `REMOTE_DATABASE_SETUP.md`
2. **Ограниченное количество подключений** - максимум 20 одновременных подключений
3. **Не запускайте другие тяжелые приложения** на этом сервере
4. **Рекомендуется настроить swap** - файл подкачки поможет при пиковых нагрузках

### Рекомендации по использованию

1. **Мониторинг ресурсов:**
   ```bash
   # Установите htop для мониторинга
   sudo apt install htop -y
   htop
   
   # Или используйте стандартные утилиты
   free -h  # Память
   df -h    # Диск
   top      # Процессы
   ```

2. **Ограничение подключений в приложении:**
   - В коде уже настроен пул соединений: `pool_size=5`, `max_overflow=5`
   - Это соответствует настройкам PostgreSQL на сервере (`max_connections=20`)

3. **Оптимизация запросов:**
   - Избегайте сложных запросов, которые требуют много памяти
   - Используйте индексы для ускорения запросов
   - Регулярно выполняйте `VACUUM` для очистки БД

4. **Регулярное обслуживание:**
   ```sql
   -- Очистка базы данных
   VACUUM ANALYZE;
   
   -- Проверка размера базы данных
   SELECT pg_size_pretty(pg_database_size('finance_db'));
   ```

## Производительность

При такой конфигурации сервер подходит для:
- ✅ Небольших и средних проектов
- ✅ До 20 одновременных пользователей
- ✅ Базы данных до нескольких ГБ

Не рекомендуется для:
- ❌ Высоконагруженных приложений
- ❌ Больших объемов данных (>5ГБ)
- ❌ Сложных аналитических запросов

## Масштабирование

Если проект вырастет, рассмотрите:
1. Увеличение RAM до 1-2ГБ
2. Использование SSD вместо HDD
3. Настройку репликации для чтения
4. Разделение на несколько серверов

## Мониторинг

Регулярно проверяйте:
- Использование памяти: `free -h`
- Использование диска: `df -h`
- Активные подключения к БД: `SELECT count(*) FROM pg_stat_activity;`
- Размер базы данных: `SELECT pg_size_pretty(pg_database_size('finance_db'));`
